HashMap

2017-12-09 Java Java 0 Comments Word Count: 2,251(words) Read Count: 10(minutes)

概念与简介

哈希表（HashMap） 是常用的Java集合之一，是继承于 AbstractMap 的对基于哈希表的 Map 接口的实现，应用场景极其丰富，许多缓存技术（比如Memcache）的核心其实就是在内存中维护一大张哈希表。

数据结构

HashMap 底层是一个哈希数组，每个元素为一个Entry，通过Key的hashCode来计算hash值，并通过hash值确定entry在哈希数组中的位置，当hash值相同时，采用的是“拉链法”来解决冲突。

总的来说就是 数组 + 链表（红黑树 jdk8+）的形式来存储数据。

方法及对象

HashMap 继承于 AbstractMap 同时实现了 Map 接口，这里只讲下比较重要的几个方法。

同时，相对于1.7，1.8在解决hash冲突时有了比较大的变化，当量表长度大于阈值 8 (TREEIFY_THRESHOLD) 个时，会将量表转化为红黑树，以减少搜索时间。原本Map.Entry接口的实现类Entry改为了Node，转化为红黑树时改用另一种实现TreeNode。

主要代码

put方法

  1. 支持空null key、null value。
  2. 利用key的hash值确定桶的位置，同时利用key的hash值和equal方法找到已存在的节点并重写返回。
  3. 新增节点时要判断桶中元素是否超过阈值，否则需要进行扩容.
  4. 同时当前节点为链表节点时，要判断是否超过链表节点阈值，超过则需要将链表转化为红黑树。

get方法

通过key的hash值找到桶的位置，并通过key的equal方法找到具体节点(Entry)。
